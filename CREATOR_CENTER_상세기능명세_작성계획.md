## 3. 정보 구조 (IA) — 페이지 단위 작업 단위

스크린샷·페이지 제목 기준으로 확인된 **크리에이터 센터 앱 영역** 내비게이션은 아래 4개로 가정한다. (이름은 와이어에 표기된 그대로 사용.)

| 순서 | 페이지 ID | 내비 라벨 (확인됨) | 명세 작성 우선순위 메모 |
|------|-----------|-------------------|-------------------------|
| P0 | `shell` | (전역) | 레이아웃·공통 헤더/푸터·로그인 CTA 구분 |
| P1 | `home-curation` | 홈 | **홈 큐레이션** 화면 — 와이어가 가장 상세히 관측됨 |
| P2 | `post-mgmt` | 포스트 관리 | 프레임별 Figma에서 재확인 필요 |
| P3 | `membership` | 멤버십 상품 | 동일 |
| P4 | `profile` | 프로필 수정 | 동일 |

Figma 상단의 **파일명·가입 버튼·도움말** 등은 제품 기능인지 Figma 호스트 UI인지 명세에서 **한 번 구분**할 것 (와이어가 “관리자 앱”만 담는 경우 제품 범위에서 제외 가능).

## 4. 공통 작성 규칙 (모든 페이지에 동일 적용)

각 페이지 명세에 최소 포함할 블록:

1. **목적·진입 경로** (어디서 이 화면으로 오는지)
2. **구성 요소 목록** (카드, 테이블, 버튼, 토글, 배지 등)
3. **사용자 액션 → 시스템 반응** (클릭, 드래그, 저장 등)
4. **상태** (로딩, 빈 목록, 권한 없음, 저장 성공/실패 등 — 와이어에 없으면 `TBD` 표기)
5. **검증·제약** (필드 길이, 이미지 규격 등 — 와이어에 적힌 수치는 반드시 인용)
6. **오픈 이슈** (와이어와 실제 정책 불일치 시 질문 목록)

## 5. 페이지별 실행 계획 (명세 “작성” 작업 단계)

아래 **Phase**는 코드 구현이 아니라 **명세 문서를 채워 나가는 순서**이다.

### CP-GLOBAL — 전역 셸 (`shell`)

| 단계 | 작업 | 완료 기준 |
|------|------|-----------|
| G-1 | Figma에서 **전체 데스크톱 프레임** 1장 캡처, 제품 영역 vs Figma 크롬 구분 표시 | 부록 또는 명세 1절에 삽입 가능 |
| G-2 | 좌측 **Creator Center** 브랜딩, 사이드바 4항목, 선택 상태(홈 강조) 규칙 기술 | 내비 전환 시 어떤 영역이 바뀌는지 문장으로 정리 |
| G-3 | (와이어에 있는 경우) 상단 **미리보기·저장** 등이 “전 페이지 공통”인지 “홈 전용”인지 프레임 비교로 확정 | 표로 “공통 / 페이지 한정” 구분 |

**체크포인트 ID:** `CP-GLOBAL-DONE` — G-1~G-3 완료 후 로그에 기록.

---

### CP-HOME — 홈 · 홈 큐레이션 (`home-curation`)

와이어에서 **관측된** 내용을 명세 초안의 근거로 삼는다.

| 단계 | 작업 | 완료 기준 |
|------|------|-----------|
| H-1 | 상태 배너 **「홈페이지 공개 중」** 의미(읽기 전용 알림 vs 토글 결과) 정의 | 비즈니스 규칙 1~2문장 또는 TBD |
| H-2 | **홈 커버 이미지** 카드: 권장 해상도 `1440 x 400px` 인용, 업로드·교체·삭제 플로우 초안 | 액션 목록 + 실패 시 TBD |
| H-3 | **고정 포스트**: PIN 배지, 썸네일·제목·**전체 공개** 표기, **해제** 버튼 동작 | 고정 1개 제한 여부 등 정책은 스테이크홀더 확인 항목으로 분리 |
| H-4 | **큐레이션 목록** 헤더: `3개` 배지 의미(그룹 수 vs 아이템 수) 프레임/주석으로 확정 | 명세에 정의 문장 확정 |
| H-5 | **큐레이션 추가** 버튼: 눌렀을 때 모달/페이지/드로어 중 무엇인지 프로토타입 따라가며 기술 | 없으면 TBD + 질문 |
| H-6 | 그룹 **「인기 콘텐츠」**: 그룹 단위 **공개** 토글, 케밥(⋯) 메뉴 항목 나열 | 메뉴 항목이 안 보이면 Figma 레이어 열람 |
| H-7 | 행 단위: 제목, **전체 공개 / VIP 구독** 등 가시성 라벨(**구독** 통일, O-11), **공개** 배지, **드래그 핸들** | 드래그 시 순서 저장 타이밍(즉시 vs 저장 버튼) 확정 |
| H-8 | **미리보기**, **변경사항 저장**: 저장 전 검증, 미리보기 대상 URL/탭 가정 | TBD 줄 수 있으나 “버튼 역할”은 한 줄 이상 |

**체크포인트 ID:** `CP-HOME-DONE`
